---
name: Orval dependent query queryKey
description: Typecheck fix for Orval-generated React Query hooks used as dependent/conditional queries.
---

When calling an Orval-generated `useX(params, { query: { enabled: someCondition } })` hook where the query depends on another query's result (e.g. fetch related items only after a parent tool/entity has loaded), TypeScript fails with "Property 'queryKey' is missing" even though other generated hooks work fine without it.

**Why:** the generated `UseQueryOptions` type expects a full options object once you touch `query`, and `enabled` alone doesn't satisfy it structurally in this codegen setup.

**How to apply:** import the matching `getXQueryKey` helper (also generated by Orval) and pass it explicitly:

```ts
const params = { category: parent?.categorySlug, limit: 4 };
const { data } = useListTools(params, {
  query: { queryKey: getListToolsQueryKey(params), enabled: !!parent },
});
```
